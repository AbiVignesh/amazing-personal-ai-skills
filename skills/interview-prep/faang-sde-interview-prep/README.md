# FAANG SDE Interview Prep Skill

**Interview prep for FAANG software engineering roles. No fluff, pattern-based, company-specific.**

---

## Quick Start (5 minutes)

1. **Assess yourself:** Run the readiness assessment workflow
2. **Identify gaps:** See assessment report to find what's weak
3. **Build roadmap:** Generate personalized study plan
4. **Execute:** Follow the plan, do mocks

→ Ready to interview in 4-12 weeks depending on current level

---

## What This Skill Does

### Workflow 1: Readiness Assessment
**When:** Start of prep, before applying to companies

**What you get:**
- Honest assessment of DSA, system design, behavioral strength
- Gap analysis (what's weak, what's OK)
- 30/60/90 day improvement plan
- Clear go/no-go for applying

**Example:** "You're strong on trees, weak on graphs. Strong system design. Behavioral needs work. Ready to apply in 4 weeks if you focus on graphs + behavioral."

---

### Workflow 2: Personalized Roadmap
**When:** After assessment, before starting prep

**What you get:**
- Week-by-week study plan
- Daily schedules (what to practice when)
- Problem counts by difficulty
- Company-specific adjustments
- Mock interview schedule

**Example:** "Week 1: Learn sliding window + two pointers (5 problems). Week 2: Add graphs (7 problems). Week 3: Start system design..."

---

### Workflow 3: Mock Interview (3 Modes)

**Mode 1: Coding Mock (45 min)**
- Timed problem-solving
- Real interview conditions
- Detailed feedback on approach, speed, correctness
- Comparison to previous mocks

**Mode 2: System Design Mock (60 min)**
- Design a system end-to-end
- Feedback on architecture, tradeoffs, scalability
- Assessment against FAANG standards

**Mode 3: Behavioral Mock (30 min)**
- Practice STAR stories
- Company-specific questions
- Feedback on communication, values alignment

---

### Workflow 4: Problem Set Curation
**When:** Need focused practice on a specific pattern

**What you get:**
- 30+ curated problems organized by pattern
- Easy → Medium → Hard progression
- Problems known to come from FAANG
- Similar problems grouped together

**Example:** "Practicing sliding window? Here are 5 problems that teach the pattern, progressing in difficulty."

---

### Workflow 5: Progress Tracking
**When:** Regular check-ins during prep

**What you get:**
- Assessment of progress since last check
- Honest feedback on pace vs. interview date
- Adjusted recommendations

---

### Workflow 6: Resume Optimization
**When:** Resume is ready for review

**What you get:**
- Honest feedback on impact statements
- Specific examples of how to reframe
- Company-specific guidance
- Resume checklist before submitting

---

### Workflow 7: Behavioral Preparation
**When:** Want to prepare STAR stories and company-specific answers

**What you get:**
- Framework for answering behavioral questions
- STAR method templates
- Company-specific focus areas
- 20+ sample questions with guidance

## Common Questions

### Q: Should I learn all 15 DSA patterns?
**A:** No. Learn 1-2 per week. After 8 weeks, you'll know them all.

### Q: How many problems should I solve?
**A:** 40-50 total if 4-8 weeks prep. 80-100+ if 12 weeks. Quality > quantity.

### Q: What about system design? Is it really 50% of hiring?
**A:** For senior roles, yes. For junior, maybe 30%. Companies vary. This skill accounts for that.

### Q: Can I use this if I'm not targeting FAANG?
**A:** Mostly yes. Core DSA patterns apply everywhere. Company profiles are FAANG-specific. System design frameworks work anywhere.

### Q: How often should I do mocks?
**A:** 1-2 per week during prep. Increase to 2-3 per week in final 2 weeks.

### Q: What if I fail an interview?
**A:** Use the assessment report to understand why. Fix that weakness (improvement plan). Try again 2-3 months later.

## Avoiding Common Mistakes

❌ **Don't:**
1. Memorize 500 problems → Learn patterns instead
2. Ignore system design → Practice 1-2 systems per week
3. Skip behavioral → 30% of hiring decision
4. Do problems without timing → Time pressure is real
5. Ignore company differences → Google ≠ Amazon
6. Practice alone without feedback → Mocks are critical
7. Rush through prep → 8 weeks is better than 2 weeks

## Pro Tips

**Tip 1: Pattern Recognition**
After solving 5-10 problems, you'll start seeing patterns repeat. That's when things click.

**Tip 2: Speed Comes Last**
Week 1-2: Focus on correctness. Weeks 3-4: Add speed. By week 8, you'll be fast.

**Tip 3: Mocks = Real Interview**
Take mocks seriously. Treat them like real interviews. The stress you feel in mocks helps you prepare for real stress.

**Tip 4: Talk Out Loud**
In practice, say your approach out loud before coding. In interviews, this is critical.

**Tip 5: Company-Specific Research**
1 week before interview, dive deep on company profile. Understand their hiring signals.

**Tip 6: Behavioral Stories**
Prepare 4+ stories that work for multiple questions. Adapt them based on what's asked.

**Tip 7: Resume Matters**
Strong resume = phone screen = chance to interview. Spend time on it.

## Getting Help

### If you're stuck on a pattern:
1. Read DSA pattern map for that pattern
2. Code the template
3. Solve 2-3 easy problems
4. Read discussion/solutions

### If you're not improving:
1. Use improvement-plan template
2. Identify root cause
3. Do targeted practice (Phase 2-4)
4. Re-test after 1 week

### If you're running out of time:
1. Focus on medium problems (best ROI)
2. Skip hard problems (diminishing returns)
3. Practice mocks more (learn from mistakes)
4. Reduce system design (focus on DSA)
