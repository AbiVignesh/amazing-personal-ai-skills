## The Improvement Plan

### Phase 1: Understand the Weakness (1-2 days)

**What to do:**
- [ ] Review 3-5 problems in this area
- [ ] Identify what you got wrong (not just the answer)
- [ ] Understand the pattern or concept

**Execution:**
1. Pick an easy problem in the weak area
2. Solve it from scratch (no hints)
3. Review the solution
4. Ask: What did I miss? Why did I fail?
5. Repeat for 2-3 more problems

**Time:** 3-4 hours total

**Success metric:** Can articulate why you were failing

---

### Phase 2: Learn / Relearn the Framework (2-3 days)

**What to do:**
- [ ] Study the pattern or concept
- [ ] Code from scratch (template-based)
- [ ] Understand intuition, not just memorize

**Execution:**

**If it's a pattern (sliding window, graphs, etc):**
1. Read DSA pattern map section
2. Code the template from memory
3. Apply to 2 problems
4. Understand when to use this pattern

**If it's a concept (complexity analysis, recursion, etc):**
1. Read explanation
2. Work through 2-3 examples by hand
3. Code solution
4. Test with edge cases

**Time:** 3-5 hours

**Success metric:** Can code the pattern/concept from scratch

---

### Phase 3: Targeted Practice (5-7 days)

**What to do:**
- [ ] Solve 10 problems in this area
- [ ] Progress: 2-3 easy, 5-7 medium, 0-2 hard
- [ ] Track time and correctness

**Execution:**

**Day 1-2: Easy problems (3 total)**
- Solve 1-2 easy problems
- Time: 10-15 min each
- Goal: Warm up, build confidence

**Day 3-5: Medium problems (5 total)**
- Solve medium problems
- Time: 30-40 min each
- Goal: Develop speed and accuracy

**Day 6-7: Speed + accuracy (2 problems)**
- Take 2 new medium problems
- Timed: 35 min each
- Goal: Real interview simulation

**Tracking:**
```
Problem | Easy/Med/Hard | Time | Correct? | Notes
--------|---------------|------|----------|-------
1       | Easy          | 12m  | Yes      | Felt confident
2       | Easy          | 14m  | Yes      | Overthought
3       | Med           | 38m  | Yes      | Had to think hard
...
```

**Time commitment:** 5 hours/day for 1 week = 35 hours total

---

### Phase 4: Speed Drills (3-5 days)

**What to do:**
- [ ] Solve 5-10 problems you've already done
- [ ] Try to beat your previous time

**Execution:**
1. Pick 5 problems you solved before (but haven't practiced recently)
2. Solve each in strict time limit
3. Don't look at your old solution first
4. Goal: Get faster without losing correctness

**Example:**
- Week 1: Solved tree BFS problem in 35 min
- Week 2 speed drill: Same problem in 20 min target

**Time commitment:** 1-2 hours total

---

### Phase 5: Mock Interview (1 session)

**What to do:**
- [ ] Take a full timed mock in your weak area
- [ ] Score yourself
- [ ] Compare to baseline

**Execution:**
1. Pick a problem you haven't seen in your weak area
2. Time yourself (realistic interview time)
3. Code it without hints
4. Score: Correctness + time + communication
5. Compare to your baseline

**Success metric:** Score improved from baseline

## Contingency: If Still Not Improving

**If after 1 week of Phase 1-3, you're still struggling:**

1. **Root cause might be deeper**
   - Example: You don't understand recursion (not just a pattern)
   - Fix: Go back and study fundamentals
   - Time: Extra 2-3 days

2. **Not enough practice volume**
   - If you did 10 problems and still struggling: Do 20 problems
   - Extend timeline

3. **Approach is wrong**
   - You might be solving problems the hard way
   - Fix: Study optimal solution approach for 3-5 problems
   - Then retry

4. **Confidence/mental block**
   - You know the pattern but panic in interview
   - Fix: Build confidence through easy wins (do 5 easy problems)
   - Then retry medium
