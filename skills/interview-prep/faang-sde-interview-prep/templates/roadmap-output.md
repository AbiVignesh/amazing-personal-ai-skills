## Week-by-Week Breakdown

### Week 1: Foundations + Speed Baseline

**Goal:** Establish baseline, start core patterns

**Daily Schedule (Mon-Fri): 2-3 hours**

**Monday:** Sliding Window Pattern
- [ ] Read DSA pattern map (sliding window section)
- [ ] Solve: Max consecutive ones, move zeroes
- [ ] Code and test edge cases
- Time: 2 hours

**Tuesday:** Continue Sliding Window
- [ ] Solve: Longest substring without repeating
- [ ] Optimize if possible
- Time: 2.5 hours

**Wednesday:** Two Pointers Pattern
- [ ] Learn two pointers framework
- [ ] Solve: Two sum (sorted), container with most water
- Time: 2.5 hours

**Thursday:** Two Pointers + Speed
- [ ] Solve: Valid palindrome
- [ ] Speed drill on Mon-Wed problems (beat your time)
- Time: 2.5 hours

**Friday:** Fast & Slow Pointers + Mock
- [ ] Learn fast & slow pattern
- [ ] Solve: Linked list cycle
- Time: 2.5 hours

**Weekend:** First Mock
- [ ] Timed coding round (45 min)
- [ ] Problem: Medium difficulty (not yet practiced)
- [ ] Score: Correctness + time
- [ ] Review: What was hard?

**Milestone Checklist:**
- [ ] 5 problems solved and understood
- [ ] Can explain each pattern
- [ ] First mock completed
- [ ] Baseline established

**Expected Speed:** Easy in 20 min, medium in 50 min (slow, that's okay)

---

### Week 2: Add Patterns 4-6 + Speed Drills

**Goal:** Master 3 more patterns, increase speed

**Daily Schedule (Mon-Fri): 2.5-3 hours**

**Monday-Tuesday:** Merge Intervals + Cyclic Sort
- [ ] Learn merge intervals (sorting + comparison)
- [ ] Solve: Merge intervals, insert interval
- [ ] Learn cyclic sort (place in correct index)
- [ ] Solve: Find missing number, find duplicate
- Time: 2.5 hours each

**Wednesday-Thursday:** In-Place Reversal + Speed
- [ ] Learn in-place reversal (pointers, swapping)
- [ ] Solve: Reverse linked list, reverse between
- [ ] Speed drills on weeks 1-2 patterns
- Time: 2.5 hours each

**Friday:** Review + Lightweight
- [ ] Do 2 speed drills (pick any week 1-2 pattern)
- [ ] Try to beat your record time
- Time: 2 hours

**Weekend:** 2 Mocks
- [ ] Mock 1: Medium problem (45 min)
- [ ] Mock 2: Easy problem speedrun (15 min)
- [ ] Review timing and correctness

**Milestone Checklist:**
- [ ] 6 new patterns mastered (3 from week 1, 3 from week 2)
- [ ] Easy problems: <15 min
- [ ] Medium problems: 35-45 min
- [ ] 2 mocks completed

---

### Week 3: Trees/Graphs + Light System Design

**Goal:** Graph/tree patterns, start system design

**Daily (Mon-Fri): 2.5 hours**

**Monday-Tuesday:** Tree Patterns (BFS/DFS)
- [ ] Learn tree BFS (level-order with queue)
- [ ] Solve: Tree level order, minimum depth
- [ ] Learn tree DFS (recursive traversal)
- [ ] Solve: Path sum, invert tree
- Time: 2.5 hours each

**Wednesday:** Graph Patterns
- [ ] Learn graph BFS (connected components)
- [ ] Solve: Number of islands
- Time: 2.5 hours

**Thursday:** Graph DFS + System Design Intro
- [ ] Learn graph DFS
- [ ] Solve: Evaluate division
- [ ] Read system design frameworks (1 hour light reading)
- Time: 2.5 hours

**Friday:** Light + Mocks
- [ ] Speed drill (30 min)
- [ ] Reread system design frameworks (1 hour)
- Time: 1.5 hours

**Weekend:** 2 Mocks
- [ ] Coding mock: Hard problem (50 min)
- [ ] System design mock: Small scope (Design a Cache, 30 min)
- [ ] Review: What's hard about system design?

**Milestone Checklist:**
- [ ] Tree patterns solid
- [ ] Graph patterns solid
- [ ] System design frameworks understood (not memorized)
- [ ] Hard DSA problem in 50 min

---

### Week 4: Company Prep + Final Polish

**Goal:** Company-specific prep, do real mocks, start applying

**Daily (Mon-Fri): 2-2.5 hours**

**Monday:** Company Profile Study
- [ ] Pick target company
- [ ] Read company profile (hiring signals, format, values)
- [ ] Read 2-3 interview reports from Blind/LeetCode Discuss
- Time: 1.5 hours

**Tuesday:** Company-Specific Problems
- [ ] Do 2 problems known to come from target company
- [ ] Focus on approach, not just correctness
- Time: 2.5 hours

**Wed-Fri:** Full Mocks + Behavioral
- [ ] Wed: Full coding round (45 min)
- [ ] Thu: System design round (50 min) OR Behavioral (30 min)
- [ ] Fri: Last mock (45 min)
- [ ] Review each for patterns

**Weekend:** Final review + applications

**Milestone Checklist:**
- [ ] Know target company's format + expectations
- [ ] 4+ full mocks completed
- [ ] Ready to apply

## Daily Time Breakdown

**Morning (8am-10am): 2 hours**
- 1 new problem (40 min coding + explanation)
- Review solution + test edge cases (20 min)

**Afternoon (2pm-4pm): 2 hours**

**Total: 4-5 hours/day, or 20-25 hours/week**

## Weekly Progress Check

Each week, self-assess:

**Speed:**
- [ ] Easy problems: Can do in <15 min?
- [ ] Medium problems: Can do in 35-45 min?
- [ ] Hard problems: Can do in 50-60 min?

**Correctness:**
- [ ] First-pass success rate: >70%?
- [ ] Edge cases: Do you think about them?

**Communication:**
- [ ] Can explain approach before coding?
- [ ] Do you think out loud?

**System Design:**
- [ ] Can break problem into services?
- [ ] Can discuss tradeoffs?

**Action if weak in any area:** Adjust next week's focus.

---

## Contingency: Falling Behind

If you fall behind:
- Cut system design practice (can catch up later)
- Focus on DSA (foundation is critical)
- Do fewer mocks, more drilling
- Extend timeline by 1-2 weeks

If you're ahead:
- Add harder problems
- Do more system design practice
- Apply earlier (if 75%+ on mocks)

## Company-Specific Adjustments

### For Google:
- Add 10+ hard problems (weeks 5-8)
- Emphasize system design depth
- Expect detailed follow-up questions

### For Amazon:
- Spend 2 weeks on behavioral (40% of hiring)
- Study 11 leadership principles deeply
- Mock interview with behavioral focus

### For Meta:
- Focus on speed (move fast culture)
- Medium difficulty is usually enough
- Shorter prep time OK

### For Apple:
- Emphasize code quality (clean, readable)
- Do system design with quality focus
- Communication really matters
